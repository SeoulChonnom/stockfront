# 화면 와이어프레임용 UI 요구사항 문서

## 1. 문서 목적

본 문서는 디자이너가 와이어프레임 및 화면 설계를 수행할 때 필요한 화면 구조, 정보 우선순위, 상태 케이스, UX 원칙을 정의한다.

## 2. 디자인 목표

- 사용자가 **10~20초 안에 시장 전체 흐름을 파악**할 수 있어야 한다.
- 숫자 정보(대표 지수)와 서술 정보(뉴스/요약)가 명확하게 구분되어야 한다.
- 개별 기사 나열이 아니라 **이슈 단위 카드 중심**으로 설계한다.
- 과거 날짜 조회가 직관적이어야 한다.
- 데이터 누락/부분 실패 상태도 사용자에게 명확히 보여야 한다.

## 4. 공통 UX 원칙

## 4-1. 정보 우선순위

1. 시장명 + 기준일
2. 시장 한줄 요약
3. 대표 지수 카드
4. 핵심 뉴스 카드
5. 시장 해설
6. 원문 링크 목록
7. 메타 정보

## 4-2. 레이아웃 원칙

- 상단은 요약과 숫자 중심
- 중단은 핵심 뉴스 카드 중심
- 하단은 원문 링크와 메타 정보
- 모바일에서는 카드 수직 스택 구조 우선

## 4-3. 상태 노출 원칙

- 실패를 숨기지 않는다.
- “일부 데이터 누락” 같은 메시지를 명시한다.
- 사용자 혼동을 줄이기 위해 **기준일**과 **생성 시각**을 분리 표기한다.

## 5. 화면 상세 요구사항

## UI-01. 최신 시장 페이지

### 목적

사용자가 가장 최근 생성된 미국 또는 한국 시장 요약 페이지를 확인한다.

### 진입 방식

- 상단 시장 탭 선택
- 기본 진입 시 최근 페이지 자동 로드

### 레이아웃 구조

1. 글로벌 헤더
2. 시장 탭
3. 페이지 헤더
4. 시장 한줄 요약 영역
5. 대표 지수 카드 영역
6. 핵심 뉴스 카드 영역
7. 시장 해설 영역
8. 원문 기사 링크 영역
9. 메타 정보 영역

### 필수 컴포넌트

#### 1) 글로벌 헤더

| 요소               | 필수 여부 | 설명                    |
| ------------------ | --------- | ----------------------- |
| 서비스명           | Y         | 예: Market Daily Brief  |
| 시장 탭            | Y         | 미국 / 한국 전환        |
| 아카이브 이동 버튼 | Y         | 과거 날짜 페이지로 이동 |

#### 2) 페이지 헤더

| 요소        | 필수 여부 | 설명                    |
| ----------- | --------- | ----------------------- |
| 페이지 제목 | Y         | 예: 미국 증시 일간 요약 |
| 기준일      | Y         | business_date           |
| 생성 시각   | Y         | generated_at            |
| 상태 배지   | Y         | READY / PARTIAL         |

표시 예시:

- `기준일: 2026-03-17`
- `생성시각: 2026-03-18 06:12`

#### 3) 시장 한줄 요약

| 요소             | 필수 여부 | 설명                    |
| ---------------- | --------- | ----------------------- |
| headline_summary | Y         | 페이지 최상단 핵심 문장 |
| 보조 설명        | N         | 짧은 부연 문장          |

UI 원칙:

- 본문 카드보다 시각적 우선순위 높음
- 1~2줄 내 표시

#### 4) 대표 지수 카드 영역

필수 카드 수:

- 미국: 3개 이상 권장
- 한국: 2개 이상 권장

카드 요소:

| 요소                  | 필수 여부 | 설명               |
| --------------------- | --------- | ------------------ |
| 지수명                | Y         | NASDAQ, S&P 500 등 |
| 종가                  | Y         | 숫자 강조          |
| 등락값                | Y         | +120.33 / -15.22   |
| 등락률                | Y         | +0.66%             |
| 장중 고가             | Y         | high               |
| 장중 저가             | Y         | low                |
| 시각적 상승/하락 표시 | Y         | 아이콘 또는 색상   |

레이아웃:

- Desktop: 3열 또는 4열 그리드
- Mobile: 1열 스택

#### 5) 핵심 뉴스 카드 영역

필수 노출 카드 수:

- 기본 3~5개

카드 요소:

| 요소           | 필수 여부 | 설명                |
| -------------- | --------- | ------------------- |
| 대표 제목      | Y         | 핵심 이슈 제목      |
| AI 요약 2~3줄  | Y         | 핵심 내용           |
| 관련 기사 수   | Y         | 예: 관련 기사 6건   |
| 태그           | Y         | 2~4개 권장          |
| 대표 링크 버튼 | Y         | 원문 보기           |
| 상세 열기 버튼 | Y         | 관련 기사 목록 열기 |

카드 인터랙션:

- 카드 클릭 시 상세 패널 또는 상세 페이지 이동
- 대표 링크 클릭 시 새 탭 이동

#### 6) 시장 해설 영역

구성:

- 상승/하락 배경
- 주요 테마
- 다음 관전 포인트

표현 방식:

- 단락형 또는 3개 섹션 카드형
- 긴 문장은 3줄 이후 접기 가능

#### 7) 원문 기사 링크 영역

표시 방식:

- 리스트형 권장

리스트 항목 요소:

| 요소        | 필수 여부 | 설명                     |
| ----------- | --------- | ------------------------ |
| 기사 제목   | Y         | 클릭 가능                |
| 언론사명    | Y         | 예: 한국경제             |
| 발행 시각   | Y         | published_at             |
| 연결 이슈명 | N         | 어느 클러스터에 속하는지 |
| 링크 타입   | N         | 원문 / 네이버            |

#### 8) 메타 정보 영역

필수 노출:

- 수집 기사 수
- 정제 기사 수
- 핵심 이슈 수
- 마지막 업데이트 시각

## UI-02. 날짜별 시장 페이지

### 목적

특정 날짜의 시장 요약 페이지를 조회한다.

### 필수 요소

| 요소                  | 필수 여부 | 설명                 |
| --------------------- | --------- | -------------------- |
| 날짜 선택 UI          | Y         | 캘린더 또는 드롭다운 |
| 시장 탭               | Y         | 미국 / 한국          |
| 해당 날짜 페이지 로드 | Y         | 없으면 Empty State   |
| 이전/다음 날짜 버튼   | Y         | 연속 탐색 편의       |

### Empty State 문구 예시

- `해당 날짜의 페이지가 아직 생성되지 않았습니다.`
- `배치가 실패했거나 데이터가 부족할 수 있습니다.`

## UI-03. 뉴스 클러스터 상세 패널/페이지

### 목적

핵심 뉴스 카드 클릭 시 관련 기사와 상세 요약을 확인한다.

### 구성 요소

| 요소             | 필수 여부 | 설명       |
| ---------------- | --------- | ---------- |
| 클러스터 제목    | Y         | 대표 제목  |
| 상세 요약        | Y         | 3~6줄 권장 |
| 키워드 태그      | Y         | 관련 주제  |
| 관련 기사 리스트 | Y         | 3건 이상   |
| 대표 링크        | Y         | 새 탭 이동 |

관련 기사 리스트 항목:

- 기사 제목
- 언론사
- 발행 시각
- 원문 링크
- 네이버 링크(있으면)

## UI-04. 아카이브 페이지

### 목적

과거 날짜의 시장 페이지를 빠르게 탐색한다.

### 구성 요소

| 요소              | 필수 여부 | 설명                |
| ----------------- | --------- | ------------------- |
| 시장 탭           | Y         | 미국 / 한국         |
| 날짜 범위 선택    | N         | 최근 7일, 30일      |
| 날짜 리스트       | Y         | 날짜별 카드 또는 표 |
| 헤드라인 미리보기 | Y         | 한줄 요약           |
| 상태 표시         | Y         | READY / PARTIAL     |

레이아웃 예시:

- 카드 리스트형
- 표형
- 달력형은 P2

## 7. 반응형 요구사항

## Desktop

- 지수 카드: 3~4열
- 뉴스 카드: 2열 가능
- 해설 섹션: 2~3단 레이아웃 가능

## Tablet

- 지수 카드: 2열
- 뉴스 카드: 1~2열

## Mobile

- 모든 카드 1열 스택
- 상단 요약과 지수 카드 우선 노출
- 기사 링크는 아코디언 또는 접기 UI 허용

## 8. 상태별 UI 정의

## 8-1. Loading

- Skeleton UI 사용 권장
- 지수 카드 3개, 뉴스 카드 3개 skeleton

## 8-2. Empty

문구 예시:

- `해당 날짜의 데이터가 아직 준비되지 않았습니다.`

## 8-3. Partial

문구 예시:

- `일부 지수 데이터가 누락되었습니다.`
- `AI 요약이 일부 생성되지 않아 원문 링크만 제공됩니다.`

## 8-4. Error

문구 예시:

- `페이지를 불러오는 중 문제가 발생했습니다. 잠시 후 다시 시도해주세요.`

---

## 9. 디자이너 오해 방지용 핵심 규칙

1. 뉴스는 기사 하나가 아니라 **이슈 묶음 카드**를 기본 단위로 설계한다.
2. 기준일(`business_date`)과 생성 시각(`generated_at`)은 반드시 동시에 표시한다.
3. 지수 숫자는 강하게 보여주되, 페이지 최상단의 핵심 정보는 **시장 한줄 요약**이다.
4. 링크만 나열하는 화면이 되지 않도록, **요약 → 카드 → 원문 확인** 흐름을 유지한다.
5. 일부 데이터 누락 시 숨기지 말고 명확한 상태 문구를 노출한다.

## 10. 와이어프레임 텍스트 구조 예시

### 최신 시장 페이지

```text
[서비스명]                         [미국] [한국] [아카이브]

미국 증시 일간 요약
기준일: 2026-03-17 | 생성시각: 2026-03-18 06:12 | 상태: READY

[헤드라인 배너]
기술주 강세로 나스닥이 상승 마감했다.

[대표 지수 카드]
[NASDAQ] [S&P 500] [DOW JONES]

[핵심 뉴스 카드 1]
제목
요약 2~3줄
태그 / 관련 기사 수 / 원문보기 / 상세보기

[핵심 뉴스 카드 2]
...

[시장 해설]
- 상승/하락 배경
- 주요 테마
- 다음 관전 포인트

[관련 기사 링크 리스트]
기사 제목 / 언론사 / 발행시각 / 링크

[메타 정보]
수집 기사 수 / 정제 기사 수 / 핵심 이슈 수 / 마지막 업데이트 시각
```
